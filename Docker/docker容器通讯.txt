容器间单向通讯
docker容器下的虚拟ip,虚拟ip无法从外侧进行直接访问，它只能在容器内部进行任意通讯
docker的容器频繁的创建删除会经常修改ip,不利于部署维护
可以为容器给予相应的名字，连接时只需要使用映射的名字就可以进行连接


docker
	create 创建一个容器但是不运行
	run	通过镜像创建容器并启动服务
		--name <容器名称>
		--link <进行通讯的容器网络名称>
		<镜像名>:<tags标签>
		
	inspect查看容器的元数据
		<容器的id>
	network与网络相关的命令
		ls列出当前docker底层网络服务的明细
		create新建网桥
			-d <新建网络的类型bridge>
			<新建网桥的名称>
		connect <网桥名> <容器名称> 将网桥和容器之间进行绑定

通过docker inspect <容器id>查看容器的原数据，原数据的json串:
	NetWorkSettings查看网络设置地方
		IPAddress容器的虚拟地址

一个容器可以直接ping同另一个容器的ip地址
但是无法直接通过容器名区ping同
可以通过docker run --link <容器名> <镜像名>:<tags>直接通过容器名区ping同网络
但是link只能进行容器的单项通讯
利用网桥就可以实现多组容器间的通讯
docker会自动通过网桥与外部的物理网卡进行连接，一次容器可以和外界进行通讯
所有处在一个网桥的容器默认之间都是互联互通的

为什么网桥之间可以互联互通
	每建立一个网桥，计算机上就会常见对应的虚拟网卡，同一个虚拟网卡的容器就可以互联互通了
	虚拟网卡要和外部进行通讯时还需要和物理网卡进行通讯,容器<-->虚拟网卡<-->物理网卡
